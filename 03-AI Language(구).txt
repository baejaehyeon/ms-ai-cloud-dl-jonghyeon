AI Language : 텍스트를 이해하고 분석하기 위한 NLP(자연어 처리) 기능을 제공하는 클라우드 기반 서비스
* 아래는 AI Foundry에 통합되기 이전의, Language Studio에서 실습 순서

* 리소스 생성 순서
1. Azure 리소스 "language" 검색 및 "언어 서비스" 생성
2. Region은 East US, 가격 책정은 Standard 선택, 하단 "책임 있는 AI 알림" 체크 후 리소스 생성
3. 리소스로 이동 후, 개요 탭 우측 하단 "Language Studio 시작하기" 클릭하여 이동
4. Language Studio 우측 상단에서 계정 선택 및 리소스(8aixxx-language) 선택

* Document Intelligence와 다르게 각 모델별 endPoint가 동일하고, Body의 Kind로 모델을 구분한다.
Extract PII : 이름, 주소, 전화번호, 이메일, 신용카드 번호, 운전면허 번호 등 개인 식별 정보
Extract Key phrases : 문서 전체 내용을 대표하는 가장 중요한 구(Phrases)
Find Linked entities : NER로 식별된 개체에 위키피디아 등 외부 지식 기반의 고유 ID 부여
Extract named entities : 사람, 장소, 조직, 시간, 제품 등 미리 정의된 일반 명사 개체
Extract Health information : 진단, 증상, 치료, 약물, 검사 결과, 신체 부위 등 의료/임상 관련 전문 개체
Custom named entity recognition : 사용자가 데이터 학습을 통해 정의한 특정 도메인 및 산업 관련 개체


* 최근 Microsoft의 전략에 따라, Azure AI Language는 Foundry Tools의 일부로 통합되고 있음.
Azure OpenAI Foundry 플랫폼 내에서 LLM(예: GPT-4)을 사용하여 대화를 처리하는 AI 에이전트를 구축할 때, 
텍스트 요약이나 복잡한 분류와 같은 특정 전문 작업을 위해 Azure AI Language 도구를 Foundry 에이전트의 도구로 연결하여 함께 사용.

* 주요 목표 및 특징
고급 NLP 기능 제공: 최첨단 트랜스포머 모델(Transformer models)을 사용하여 복잡한 언어 분석 수행.
생성형 AI 앱 지원: 텍스트에서 정보를 추출, 분류, 요약하여 생성형 AI 앱 개발을 가속화.
다국어 지원: 하나의 언어로 모델을 학습시키고 다른 여러 언어에 사용 가능.
커스터마이징 가능: 미리 구성된 기능뿐만 아니라, 사용자 지정 데이터를 사용하여 모델을 학습시킬 수 있음.
개발 용이성: 웹 기반 Language Studio, REST API, 및 다양한 클라이언트 라이브러리(Python, .NET, Java, JavaScript 등)를 제공.

* 주요 기능
명명된 엔터티 인식 (NER) : 텍스트에서 사람, 장소, 날짜, 조직 등과 같은 명명된 엔터티를 식별하고 분류합니다.
감정 분석 및 의견 마이닝 : 텍스트의 전체적인 감정 (긍정, 부정, 중립)을 식별하고, 특정 요소(제품 특징 등)에 연결된 의견을 분석합니다.
핵심 구 추출 : 주요 주제를 나타내는 **핵심 구(Key phrases)**를 빠르게 식별합니다.
언어 감지 : 입력 텍스트가 어떤 언어로 작성되었는지 식별합니다.
요약 : 긴 문서나 대화 내용을 핵심 문장으로 압축하여 요약합니다.
PII(개인 식별 정보) 수정/제거 : 이름, 주소, 전화번호 등 개인 식별 정보를 식별하고 제거(수정/Redaction)하여 개인 정보를 보호합니다.
Text Analytics for Health : 비정형 텍스트(예: 임상 기록)에서 의료 관련 용어, 관계, 엔터티를 추출합니다.
