Azure AI Foundry : 엔터프라이즈급 AI 개발 및 배포를 위한 통합 플랫폼. 다양한 파운데이션 모델과 AI 도구들을 결합하여 복잡하고 확장 가능한 AI 애플리케이션 및 에이전트를 구축, 관리, 운영
* AI Foundry에 통합된, "대화 언어 이해" 실습

* 리소스 생성 및 실습 순서
1. Azure 리소스 "foundry" 검색 및 생성
2. Region은 East US 2 선택 후 리소스 생성
3. 리소스로 이동 후, 개요 탭 "Go to Foundry Portal" 클릭하여 이동
4. 좌측 "미세 조정" 탭 클릭 => 화면 중앙 "AI 서비스 미세조정" 클릭 => "+ 미세 조정" 클릭
5. "대화 언어 이해" 클릭 => "기존 프로젝트 가져오기" => 라벨링 json 파일 업로드(Ex. CLU-EmailAppDemo.json)
6. 모델 학습 => 모델 평가 => 모델 배포 순차 진행

* 좌측 "스키마"와 "데이터 관리"에, 업로드한 JSON 구조에 따라 모델이 학습할 "의도", "발화", "언어"가 구분되어 있음.
- 의도 : JSON의 intents의 값들이 "의도"로 들어가 있음. "의도"란, "발화(사용자 텍스트)"를 보고 어떠한 행위 or 목적인지 구분해 놓은 것!
- 발화 : 사용자가 실제로 입력하거나 말하는 텍스트 문장 그 자체
- 엔터티 : 발화의 중요한 정보 조각. 
  ex 발화 => 이번주 금요일 오후 7시 영화 티켓 예매해줘.
  ex 엔터티 => Date: "이번 주 금요일" / Time: "오후 7시" / Item: "영화 티켓"
  
* 주요 기능
모델 카탈로그 및 배포 : 다양한 오픈 소스 및 독점 파운데이션 모델을 찾아 쉽게 배포하고 엔드포인트로 관리.
프롬프트 흐름 : LLM 기반 애플리케이션의 개발, 테스트, 평가를 위한 시각적 워크플로우를 제공. RAG 등 복잡한 체인을 쉽게 구성.
RAG(검색 증강 생성) : 외부 데이터 소스(문서, DB 등)를 LLM에 연결하여 답변의 정확성과 최신성을 높임.
모델 미세 조정(Fine-Tuning) : 사용자 지정 데이터로 모델을 학습시켜 특정 작업에 대한 성능을 최적화.
콘텐츠 안전 필터 : 폭력, 혐오 발언, 성적인 내용 등 유해하거나 부적절한 콘텐츠를 감지하고 필터링하는 기능을 제공.
평가 및 모니터링 : 구축된 AI 솔루션의 성능, 안전성, 편향성 등을 평가하고 운영 환경에서 모니터링.
Azure AI Services 통합 : Language, Vision, Speech 등 다른 Azure Cognitive Services(이제 Azure AI Services로 통칭)를 LLM 에이전트의 **도구(Tool)**로 통합하여 사용.
